HTML5 新特性

  1. 新的结构元素标签
    语义化标签
  1. HTML5 内建了对音频和视频的支持，使得网页能够直接嵌入媒体内容，无需额外的插件。
    controls 是一个属性，通常用于 < audio > 和 < video > 元素中，表示是否显示浏览器默认的控制界面（如播放/暂停按钮、音量调节、进度条等）。这个属性是布尔属性，只需在 < audio> 或 < video> 标签中添加 controls，即可启用浏览器提供的默认控件。
  1. HTML5 提供了两种本地存储机制，允许网页在用户的浏览器中存储数据，提供更好的离线体验。
    localStorage：用于存储数据在用户浏览器中，数据不会随着浏览器关闭而丢失。
    sessionStorage：用于存储数据在一个会话中，当浏览器标签页关闭时数据会丢失。
  1. HTML5 引入了 < canvas> 元素，它允许通过 JavaScript 在网页上动态绘制图形、动画和其他图像效果。常用于游戏、数据可视化和图形编辑器等场景。
  1. 地理定位（Geolocation API），使得网页能够获取用户的位置，适用于地图、定位服务等应用。
  1. 表单新特性
    HTML5 对表单做了很多增强，增加了新的输入类型和属性：新输入类型：email、tel、url、number、date、time 等。
    < input type=”email”>：自动验证输入的电子邮件地址格式。
    < input type=”date”>：提供日期选择器。
    < input type=”number”>：限制只能输入数字。
    < input type=”range”>：创建一个滑动条。
    < textarea>：支持自定义行数和列数的文本输入框。
  1. WebSockets 提供了一个全双工的通信通道，允许浏览器和服务器之间进行实时、低延迟的通信。适用于实时聊天、在线游戏和股票价格更新等应用。
  1. 拖放 API（Drag and Drop API）
    HTML5 提供了拖放 API，允许用户通过拖动文件、文本或其他元素，将它们拖入网页进行操作。
  1. SVG 支持
    HTML5 原生支持 SVG（可缩放矢量图形），使得网页能够嵌入和显示高质量的图形，而无需使用图片文件。它使得图像在任何尺寸下都不会失真。
  1. Web Workers 允许在浏览器后台线程中运行 JavaScript 代码，从而不会阻塞主线程，提升了性能。适用于处理大量数据、进行计算密集型操作等。

BEM设计模式

BEM (Block, Element, Modifier)
通过系统化的命名方式解决复杂 UI 组件样式的组织和管理问题。

  • BEM 将页面分解为以下三个部分：
    • Block（块）：
      表示一个独立的功能模块或组件，页面中的核心构建块。
      例子：header, menu, button, form。
    • Element（元素）：
      表示块中的组成部分，依赖于其所在的块。
      例子：menu__item, button__icon, form__input。
      使用双下划线 __ 连接。
    • Modifier（修饰符）：
      表示块或元素的外观、状态或行为的变体。
      例子：button–disabled, menu__item–active。
      使用双连字符 – 连接。

弹性盒flex

组成部分：弹性容器、弹性盒子、主轴、侧轴

Flex 是 Flexible Box 的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。注意，设为 Flex 布局以后，子元素的float、clear和vertical-align属性将失效。
任何一个容器都可以指定为 Flex 布局，行内元素也可以。Webkit 内核的浏览器，必须加上-webkit前缀。

容器默认存在两根轴：水平的主轴（main axis）和垂直的轴（cross axis）。
最左边：主轴的开始位置（与边框的交叉点）叫做main start，最右边的结束位置叫做main end；
垂直轴的开始位置叫做cross start，结束位置叫做cross end。
项目默认沿水平轴排列。
（单个项目占据的主轴空间）项目宽：main size，项目高：cross size。

网格grid

与 Flexbox 主要处理一维布局不同，Grid 布局可以同时控制 行（rows） 和 列（columns），因此适用于更复杂的页面结构。（fr 是一种相对单位，表示容器中可用空间的比例）

  1. display: grid / display: inline-grid
    display: grid;：使元素成为网格容器。
    display: inline-grid;：使元素成为内联网格容器（它会表现得像内联元素，但依然拥有网格布局特性）。
  2. grid-template-columns 和 grid-template-rows
    grid-template-columns：定义网格容器的列（列宽）。
    grid-template-rows：定义网格容器的行（行高）。
    它们都可以使用长度单位（px, em, %）和 fr 单位，fr 是一个比例单位，表示可用空间的分配。
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
    使用 auto-fill 或 auto-fit 配合 minmax()，使得网格项在容器内自动填充并响应式调整。
  3. grid-gap 或 gap
    用来设置网格项之间的间距。gap 是简写，grid-gap 和 gap 是等效的。
    示例：gap: 20px; 表示网格项之间的间距是 20px。
  4. grid-column 和 grid-row
    用于指定一个网格项（子元素）跨越的列和行数。
    grid-column: span 2; 让网格项跨越两列。
    grid-row: span 2; 让网格项跨越两行。
  5. grid-template-areas
    允许使用区域命名来更直观地安排网格项的位置。
    可以通过区域名称为网格项分配位置，适合于更复杂的布局。
  6. 对齐
    place-items 是 align-items 和 justify-items 的简写，用于同时设置水平和垂直对齐方式。
  1. 容器（Grid Container）：通过将 display: grid; 应用于一个元素，使它成为一个网格容器。
  2. 项目（Grid Items）：网格容器中的直接子元素，它们会被自动放置到网格中。

响应式

响应式布局（Responsive Layout）是指通过 CSS 使得网页能够根据不同的设备屏幕大小和分辨率自动调整布局，使页面内容在手机、平板、桌面等设备上都能良好展示。实现响应式布局的主要方法是通过 媒体查询（Media Queries）来为不同屏幕尺寸设置不同的样式。

  • 视口（Viewport）：
    为了确保页面在不同设备上以正确的比例显示，使用 < meta> 标签来设置视口（viewport）是很重要的。

百分比宽度和视口单位（vw, vh）：
使用百分比（%）和视口单位（vw 和 vh）来实现布局的自适应。
vw（视口宽度）：相对于视口宽度的 1%。
vh（视口高度）：相对于视口高度的 1%。

  • 采用灵活布局flex,grid
  • *响应式图片和文字大小
    使用 srcset 和 sizes 属性来为不同的屏幕分辨率和设备尺寸提供不同大小的图片，以减少加载时间。
    srcset：定义不同分辨率的图片文件。
    sizes：根据屏幕宽度指定图片显示的尺寸。

使用相对单位（如 em, rem, %）或视口单位（如 vw, vh）来设置字体大小，使得字体能够随屏幕大小变化。
em：相对于当前元素的字体大小计算。
rem：相对于根元素（html）的字体大小计算，通常是响应式设计中更常用的单位。目前rem布局方案中，将网页等分成10份， HTML标签的字号为视口宽度的 1/10。

使用flexible.js配合rem实现在不同宽度的设备中，网页元素尺寸等比缩放效果。flexible.js是手淘开发出的一个用来适配移动端的js框架。核心原理就是根据不同的视口宽度给网页中html根节点设置不同的font-size。

vw：viewport width
Ø 1vw = 1/100视口宽度
l vh：viewport height
Ø 1vh = 1/100视口高度
元素的尺寸 = px单位数值 / ( 1/100 视口宽宽度 )
