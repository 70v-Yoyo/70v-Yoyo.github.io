DOM 生命周期

  1. 当 DOM 准备就绪时，document 上的 DOMContentLoaded 事件就会被触发。在这个阶段，我们可以将 JavaScript 应用于元素。
  2. 诸如 <script>...</script> 或 <script src="..."></script> 之类的脚本会阻塞 DOMContentLoaded，浏览器将等待它们执行结束。
  3. 图片和其他资源仍然可以继续被加载。
  4. 当页面和所有资源都加载完成时，window 上的 load 事件就会被触发。我们很少使用它，因为通常无需等待那么长时间。
  5. 当用户想要离开页面时，window 上的 beforeunload 事件就会被触发。如果我们取消这个事件，浏览器就会询问我们是否真的要离开（例如，我们有未保存的更改）。
  6. 当用户最终离开时，window 上的 unload 事件就会被触发。在处理程序中，我们只能执行不涉及延迟或询问用户的简单操作。正是由于这个限制，它很少被使用。我们可以使用 navigator.sendBeacon 来发送网络请求。

HTML5 新特性

  1. HTML5 内建了对音频和视频的支持，使得网页能够直接嵌入媒体内容，无需额外的插件。
    controls 是一个属性，通常用于 < audio > 和 < video > 元素中，表示是否显示浏览器默认的控制界面（如播放/暂停按钮、音量调节、进度条等）。这个属性是布尔属性，只需在 < audio> 或 < video> 标签中添加 controls，即可启用浏览器提供的默认控件。
  1. HTML5 提供两种本地存储机制，允许网页在用户的浏览器中存储数据，提供更好的离线体验。
    localStorage：用于存储数据在用户浏览器中，数据不会随着浏览器关闭而丢失。
    sessionStorage：用于存储数据在一个会话中，当浏览器标签页关闭时数据会丢失。
  1. HTML5 引入了 < canvas> 元素，它允许通过 JavaScript 在网页上动态绘制图形、动画和其他图像效果。常用于游戏、数据可视化和图形编辑器等场景。
  1. 地理定位（Geolocation API），使得网页能够获取用户的位置，适用于地图、定位服务等应用。
  1. WebSockets 提供一个全双工通信通道，允许浏览器和服务器之间进行实时、低延迟的通信。适用于实时聊天、在线游戏、股票价格更新等应用。
  1. 拖放 API（Drag and Drop API）
    HTML5 提供拖放 API，允许用户通过拖动文件、文本或其他元素，将它们拖入网页进行操作。
  1. SVG 支持
    HTML5 原生支持 SVG（可缩放矢量图形），使得网页能嵌入和显示高质量的图形，而无需使用图片文件。它使得图像在任何尺寸下都不会失真。
  1. Web Workers 允许浏览器后台线程中运行 JavaScript 代码，不会阻塞主线程，提升性能。适用处理大量数据、计算密集型操作等。
  1. classList属性是HTML5新增的一个属性，返回元素的类名。ie10以上版本支持。该属性用于在元素中添加，移除及切换 CSS 类。

  • BFC（Block Formatting Context）
    BFC（块级格式化上下文）是 CSS 布局概念，决定元素如何定位及其子元素如何排列。
    理解 BFC 是解决许多常见布局问题（如清除浮动、避免重叠等）的关键。
    • BFC规则：6条
      内部box在垂直方向一个接一个放置
      box垂直方向距离由margin决定，属于同一个bfc的两个相邻box的margin会重叠
      对于从左到右的格式化，每个元素margin box的左边，与包含块border box左边相接触，即使浮动也如此。否则相反。
      BFC区域不和float元素重叠
      内外容器元素互不影响
      计算BFC高度时浮动元素也参与计算
    • 哪些元素生成BFC？
      根元素
      float属性不为none
      position为absolute/fixed
      display为inline-block
      overflow不为visible
    • 清除浮动带来的影响：【防止塌陷】
      1.给父元素设置固定宽度
      2.在最后一个浮动元素后面加一个空块级元素，设置clear:both；
      3.伪元素选择器清除
      4.overflow:hidden;默认是visible; scroll出现滚动条为auto;inherit;当文本超出块盒的效果。
      那这跟float有什么关系？除了overflow:visible，其他overflow显示出BFC区域 block formatting context，其计算BFC高度时浮动元素也参与计算
      常见的formatting context有block formatting context, inline formatting context （IFC）决定定位和排版
      BFC只有block-level box参与，BFC 是布局隔离环境，内部与外部互不干扰

多端交互体验

  • touch-action: manipulation; 是 CSS 的一个属性，用于优化触摸屏设备上的交互体验，特别是在 移动端。
    • 主要作用：
      加快按钮的点击响应速度
      在移动设备上，默认点击按钮时，浏览器会等待 300ms，以确认用户是否要进行 双击缩放。
      touch-action: manipulation; 禁用了双击缩放，立即响应点击事件，减少 点击延迟（类似 fastClick.js 的效果）。
      防止不必要的浏览器手势（如双指缩放）
      允许 滚动，但禁用双指缩放和双击缩放，减少误操作。
    • touch-action` 的其他取值
值 作用
auto 浏览器默认行为（允许滚动、缩放、双击等）。
none 禁止所有触摸手势（滚动、缩放等）。
manipulation 允许滚动和拖动，但禁用双击缩放（⚡ 推荐）。
pan-x 允许 水平滚动，禁用垂直滚动。
pan-y 允许 垂直滚动，禁用水平滚动。

BEM设计模式

  • 定义：BEM (Block, Element, Modifier)
    BEM 将页面分解为以下三个部分：
    • Block（块）：
      表示一个独立的功能模块或组件，页面中的核心构建块。
      例子：header, menu, button, form。
    • Element（元素）：
      表示块中的组成部分，依赖于其所在的块。
      例子：menu__item, button__icon, form__input。
      使用双下划线 __ 连接。
    • Modifier（修饰符）：
      表示块或元素的外观、状态或行为的变体。
      例子：button–disabled, menu__item–active。
      使用双连字符 – 连接。
  • 作用：系统化的命名方式解决复杂 UI 组件样式的组织管理问题。

sprite v.s. iconfont 性能对比

维度 雪碧图 iconfont
请求数量 ✅ 一个图 ✅ 一个字体文件
文件大小 ❌ 大（图片合成） ✅ 小（通常几十 KB）
加载速度 ⭕ 接近 ⭕ 接近
可维护性 ❌ 低 ✅ 高
响应式/样式控制 ❌ 差 ✅ 强（支持变色、缩放）
高清适配 ❌ 需要多图或SVG ✅ 自带矢量支持
  • 雪碧图是不是已经被淘汰？没完全淘汰，但基本被「现代替代方案」边缘化了。雪碧图曾经辉煌一时，是因为：
    在 HTTP/1.1 时代，每个小图片都需要一次请求，于是大家把几十个图标合成一张图 → 雪碧图（sprite），大幅优化页面加载性能。
    但现在，情况变了：
演变 技术变化
✅ HTTP/2 / HTTP/3 并发请求优化了，加载多个图标不再慢
✅ SVG 图标 可缩放、可变色、结构轻、语义强
✅ iconfont 字体方式加载图标，灵活控制大小颜色
✅ 图标组件库（React/Vue） SVG icon 支持组件化加载
✅ Tailwind/Iconify/Unplugin-icons 等工具 动态引入图标、按需打包、自动优化

所以，雪碧图现在：
性能优势不再明显（HTTP/2 并发快）
开发维护成本高（新增图标重新合成）
样式适配性弱（无法变色/缩放，响应式差）

  • 更现代的建议方案推荐：
    🔥 SVG Sprite（symbol + <use> 引用） → 语义强、灵活控制。SVG 图标现在支持非常好，像组件一样用，控制颜色、动画、大小都很方便，性能也优秀。
    🎨 iconfont（阿里iconfont平台） → 快速上手，适合传统项目
    📦 现代图标库（如 Iconify、Unplugin-icons、Heroicons） → 自动按需加载，适合 Vue/React 项目
    🚀 组件库内置图标系统（例如 Element Plus、Ant Design）

弹性盒flex

组成部分：弹性容器、弹性盒子、主轴、侧轴

Flex 是 Flexible Box 的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。注意，设为 Flex 布局以后，子元素的float、clear和vertical-align属性将失效。
任何一个容器都可以指定为 Flex 布局，行内元素也可以。Webkit 内核的浏览器，必须加上-webkit前缀。

容器默认存在两根轴：水平的主轴（main axis）和垂直的轴（cross axis）。
最左边：主轴的开始位置（与边框的交叉点）叫做main start，最右边的结束位置叫做main end；
垂直轴的开始位置叫做cross start，结束位置叫做cross end。
项目默认沿水平轴排列。
（单个项目占据的主轴空间）项目宽：main size，项目高：cross size。

网格grid

与 Flexbox 主要处理一维布局不同，Grid 布局可以同时控制 行（rows） 和 列（columns），因此适用于更复杂的页面结构。（fr 是一种相对单位，表示容器中可用空间的比例）

  1. display: grid / display: inline-grid
    display: grid;：使元素成为网格容器。
    display: inline-grid;：使元素成为内联网格容器（它会表现得像内联元素，但依然拥有网格布局特性）。
  2. grid-template-columns 和 grid-template-rows
    grid-template-columns：定义网格容器的列（列宽）。
    grid-template-rows：定义网格容器的行（行高）。
    它们都可以使用长度单位（px, em, %）和 fr 单位，fr 是一个比例单位，表示可用空间的分配。
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
    使用 auto-fill 或 auto-fit 配合 minmax()，使得网格项在容器内自动填充并响应式调整。
  3. grid-gap 或 gap
    用来设置网格项之间的间距。gap 是简写，grid-gap 和 gap 是等效的。
    示例：gap: 20px; 表示网格项之间的间距是 20px。
  4. grid-column 和 grid-row
    用于指定一个网格项（子元素）跨越的列和行数。
    grid-column: span 2; 让网格项跨越两列。
    grid-row: span 2; 让网格项跨越两行。
  5. grid-template-areas
    允许使用区域命名来更直观地安排网格项的位置。
    可以通过区域名称为网格项分配位置，适合于更复杂的布局。
  6. 对齐
    place-items 是 align-items 和 justify-items 的简写，用于同时设置水平和垂直对齐方式。
  1. 容器（Grid Container）：通过将 display: grid; 应用于一个元素，使它成为一个网格容器。
  2. 项目（Grid Items）：网格容器中的直接子元素，它们会被自动放置到网格中。

特殊布局(圣杯/双飞翼)

圣杯和双飞翼在外观上看起来是一样的，实现方法不太一样。


  • 区别：
    圣杯的padding放在父层,利用了float布局、position定位、margin负边距
    双飞翼则将 padding放在了main内层嵌套的 div上，仅用了 float布局、margin 负边距
    貌似，看起来双飞翼css更简单，但多嵌套了一层 html 标签，也不好说哪个更好了，根据实际需要，灵活应用吧。
    那么这种自适应宽度三栏布局的优点有哪些呢，如下：
    内容与布局分离，main部分自适应宽度，容易在定宽布局和流体布局中切换
    在浏览器上的兼容性非常好，IE5.5以上都支持
    三列布局位置切换起来非常方便，只要修改他们自身的margin属性就可以
